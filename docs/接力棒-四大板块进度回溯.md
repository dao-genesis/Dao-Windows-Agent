## 〇、一句话本源（用户历轮反复定调）

以 **Devin Desktop 插件版（编程 AI 基底）+ Proxy Pro（提示词隔离替换）+ 本仓 Windows 机控**
**三插件融为一体的"归一插件"** 为核心：装进 IDE 后，**面板里直接就是整台 Windows 桌面本体**
（RDP 原生协议级路由，非截图推流），AI 在其中全权操作用户电脑、与用户日常桌面**并行不悖**；
对标 Devin Cloud——把用户电脑当作 AI 的"虚拟机"来操作，但因跑在用户真机上、集成度更高。
FreeCAD/KiCad/嘉立创/HA 等**专精领域 = 可插拔子板块**（类 VS Code 插件 / MCP），
经"模式切换"演化出 domain 专精形态，终局是**通用调度层**跨领域自主接续联动。

## 一、四大板块开发历程（循序渐进）+ 当前进度

### 板块① Windows 全接管（整机桌面级路由）—— 主战场
- **目标**：每个 IDE 窗口 = 一路独立完整 Windows 桌面会话，面板=整块桌面本体，用户主桌面零打扰。
- **已达成**：
  - 桌面级路由链路 **已真机全绿**（路线A 阶段一/二/三）：Webview(guacamole-common-js canvas)
    → WS 隧道(guacamole-lite) → guacd(Docker) → RDP → Windows 桌面；键鼠双向、剪贴板双向、
    断线重连回同会话、**多 IDE 窗口=多路 RDP 并行**（`qwinsta` 实证多会话同 Active 互不劫持）。
  - 归一主页 Windows 总控（VSIX 0.3.0）：RDP 官方五页配置收编（.json+.rdp+mstsc 启动）、
    账号盘点/建销/一键开桌面、子板块启停管理。
  - 隧道 10s 掉线循环根因修复（`maxInactivityTime:0`）；KVM/TCG 蓝屏坑已沉淀。
  - **单 IDE 窗口内多实例「分身」已落地**（VSIX 0.6.3）：`desktopHtml()` 实例注册表 `instances[]`，
    每路分身=独立 Guacamole client+独立画布+独立重连状态机；顶栏分身 tab + 「＋分身」按钮，可增开/切换/关闭；
    键盘只路由活动分身、剪贴板只对活动分身收发（多路互不串键/串贴）。
  - **单账号多路并行 RDP 已真机实证**（winlab · termsrv 10.0.26100.1 / Win11 24H2）：修好 rdpwrap 后
    同账号 `dao` 两路并发 `xfreerdp` 同时 Active（`qwinsta` 见 rdp-tcp#0 ID1 + rdp-tcp#1 ID2），
    先连的**不再被后连的踢下线**（此前报 `ERRINFO_DISCONNECTED_BY_OTHER_CONNECTION`）。每路=独立 Windows
    会话（session 边界即进程/窗口天然隔离）。
- **缺口**：guest 内 guacd/RDP 隧道尚未在冷启动脚本**一键拉起**（面板可开、链路需手动起）；
  输入法/多显/DPI 打磨待续；「分身」在 guest 内 Devin Desktop webview 里的整屏录屏演示未做
  （隧道/guacd 当前跑在宿主，guest 内 webview 触达需网络拓扑对齐）。
  - **rdpwrap 真机根因（已沉淀，firstlogon.ps1 已修）**：Win11 24H2(26100.x) 上原 `firstlogon.ps1` 装完
    `rdpwrap.dll` 恒 =no——**Defender 把 rdpwrap.dll 判 HackTool 即时清除**；且 `RDPWInst -i -s` 对
    "不受支持"版本直接拒装。修法：装前 `Add-MpPreference -ExclusionPath`；`-i -s`→`-i -o` 强制装；
    ini 刷社区版(sebaxakerhtc·含 `[10.0.26100.1]` 段)前先 `Stop-Service TermService` 解锁再覆盖。

### 板块② 三插件融为一体（AI 交互基底 + 提示词隔离 + 机控）
- **目标**：合三为一，以**模式**为枢纽（提示词覆盖 + 工具面裁剪），态持久化 `~/.dao/mode.json` 契约。
- **已达成**：
  - `ide/vscode/dao-ai-base/`（vendor 自 windsurf-assistant/plugins/dao-desktop，Cascade 三模式面板）。
  - `ide/vscode/dao-proxy-pro/`（Proxy Pro SP 隔离替换引擎薄片 + 帛书经藏文本，读 mode.json 联动）。
  - `core/agent/modes.py` 四模式：`primary`（帛书+编程全集+整机辅）/`coding`（官方原貌）/
    `windows`（整机主战场）/`domain:<app>`（专精，从 profile 现算）。
  - extension.js 启动即激活两引擎；命令面板 11 条 DAO 命令；`提示词引擎:状态`/`道化预览` 真机验证；
    test_modes + test_proxy_fusion 全绿。
- **缺口**：模式切换后已开的 Cascade 面板不主动刷新能力面（需重开）；
  dao-ai-base 的 Cascade 会话与桥 session_id 尚未互认；vendor 切片为 7-09 版，上游更新未回灌。

### 板块③ 专精领域工作层（FreeCAD / KiCad / 嘉立创 / HA）
- **目标**：每软件 = 一份 profile 薄片；`@句柄` 唤起领域层；外部子插件经描述符+RPC 收编为 external 画像。
- **已达成**：
  - 内置画像：kicad/freecad/jlceda/mspaint/notepad/system/browser 就位。
  - `/api/route` 实测 `@kicad 导出 gerber` → domain 层 + `export_gerbers` 候选命中。
  - 本机真装 KiCad pcbnew / FreeCAD 0.19 级别① invoke 实跑全通。
  - 子插件宿主 `bridge/subplugin_host.py`：一份 spec 即起真实本地 `/invoke` 端点 + 自动写描述符收编
    （首例 `bridge/subplugin_specs/homeassistant.json` 包 ha-copilot/hactl）。
  - **子板块源在各仓真实存在**：`Dao-3D-Modeling-Agent/90-归一_IDE/vscode-dao-freecad`、
    `Dao-PCB-Design-Agent/vscode-dao-kicad`、`.../vscode-dao-lceda`、`ha-copilot/ide-extension`；
    `ide/vscode/unify.js` 能把它们折入 vendor/ 合并 contributes 成**一个归一 VSIX**。
- **缺口**：**unify.js 折入四领域的完整归一 VSIX 尚未真正构建/真机装过一次**（仓里现成 vsix 是裸构建）；
