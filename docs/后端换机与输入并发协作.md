# 后端换机 与 输入并发协作（道并行而不相悖）

> 承接 `正本清源-桌面级路由本源.md`。本文记两件本源工作：
> ① 把桌面路由的「后端」从我自己的 QEMU 靶机换成**用户本地电脑**（经既有内网穿透）；
> ② 同一分身上**用户与 Agent 输入并行而不相悖**（你操作你的，我操作我的）。

---

## 一、后端换机 · TCP↔WebSocket 转发对

### 本源难点
桌面路由链路是 `Webview → 隧道(4823) → guacd(4822) → RDP`。guacd 只会拨 **原始 TCP** 到 RDP:3389。
要把"后端"从本机靶机换成用户本地电脑，而用户机只经 **DAO Bridge / CloudFlare 这类 HTTP/WS 穿透**
暴露（无公网裸 TCP），需要一对转发件把 RDP 的 TCP 字节流封进 WebSocket 穿隧道：

```
guacd ── TCP ──► connector(隧道侧 127.0.0.1:<随机口>)
                    │  ws(s)://<穿透公网URL>/rdp   (Bearer 鉴权)
                    ▼
                agent(用户机侧) ── TCP ──► 127.0.0.1:3389 (用户机 RDP)
```

- **agent**（`forward.js` · 用户机侧）：`node forward.js agent --port 19389 --token <Bearer> --target 127.0.0.1:3389`，
  由 dao-vsix 插件宿主或独立 node 拉起，经既有穿透通道暴露为公网 `wss://…/rdp`。
- **connector**（隧道侧）：铸 token 时 `resolveTarget()` 见账号带 `via` 即**惰性**起一个本地 TCP 口，
  把 RDP 目标 `hostname/port` **重写**指向该本地口；同一 `via` 全程只起一个 connector（多分身共用）。

### 账号注册表声明（`accounts.json`，已 .gitignore）
```json
{ "user-pc": { "username": "u", "password": "p",
               "via": "wss://<穿透公网URL>/rdp", "viaToken": "<Bearer>" } }
```
`hostname/port` 由 forward 层自动改写，可省略；`via/viaToken` **不会**下发进 token（服务端内解）。

### 诚实边界
- 转发只搬字节、不解 RDP；跨公网桌面交互体验取决于穿透通道 RTT/带宽（局域网/优质中继最佳）。
- 用户机需已开 RDP（或 rdpwrap 多路）。凭据留在服务端 token 加密内，穿透握手用 Bearer 鉴权。
- 同一机理同样适配「其它 IDE / 其它 Agent / Devin Cloud 经 MCP」——`via` 指向谁就换成谁的后端。

## 二、输入并发协作 · 分身级仲裁（`input-arbiter.js`）

### 两条真相
1. **不同分身 = 不同 RDP 会话 = Windows 各自独立输入队列**——「你操作你的，我操作我的」
   天然隔离（两台完全隔离的电脑）。仲裁器按 key 分桶，不同 key 永不互阻。
2. **同一分身上多操作者**（用户前端 canvas / Agent 后端 UIA·PostMessage）会在会话内交错抢占。
   故对同一 key 用**短租约轮替**：任一时刻至多一个持有者；租约带 TTL 自动过期（离场不永久占用）；
   **用户默认优先级高于 Agent**（用户随时接管/协助/纠偏），同级先到先得。

### HTTP 面（隧道 4824）
- `POST /input?op=acquire&key=<分身key>&owner=<id>&kind=human|agent[&ttl=ms]` → 申请/续租，
  返回 `{granted, holder, preempted?}`；`granted=false` 时 `holder` 供前端提示"AI 正在操作此分身"。
- `POST /input?op=release&key=…&owner=…` → 释放（仅持有者本人有效）。
- `GET /input[?key=…]` → 查持有者/列全部。

`key` 复用租约 key（`account:<名>#<slot>` / `ide_…`），与桌面路由/会话租约同一命名空间。

### 协作语义
- Agent 后端注入前先 `acquire(kind=agent)`；用户在前端操作即 `acquire(kind=human)` 抢占，
  Agent 侧 `canInput` 立即转假 → 停手，实现"用户一动手，AI 让位"。
- 所有操作成果（无论前端 canvas 还是后端 UIA）都落在**同一路真实 RDP 会话**上，
  故用户在 IDE 面板始终看得见 Agent 的每一步（后端调用·前端回显）。

### 诚实边界
仲裁是**协作层约定**（双方都遵守才有效），非 Windows 内核强制锁；对不经隧道、直接在会话内乱入的
第三方进程不设防。分身间的**硬隔离**仍由 RDP 会话边界（独立输入队列）保证。
